## 배경
카카오 로그인 직후 `/today` 첫 진입 체감 개선(P0) 및 `auth-required` 로딩 화면 UI 통일 작업입니다. (web only, 배포 없음)

## 재현
1. 비로그인 상태에서 `/today` 또는 `/friends` 진입
2. 인증 가드가 동작하면서 `/auth`로 이동
3. 카카오 로그인 완료 후 `/today`로 리다이렉트
4. 기존에는 로그인 직후 첫 진입 피드백이 없어 전환 체감이 약함

## 원인
- 로그인 성공 직후 상태를 `/today` 화면으로 전달/소비하는 UX 상태값이 없었음
- 인증 가드 로딩 UI가 단순 텍스트로, `/auth` 카드 UI와 톤앤매너가 분리되어 있었음

## 해결
- 로그인 성공 시 `sessionStorage`에 1회성 플래그(`routine-auth-just-signed-in`) 저장
- `/today` 첫 진입 시 플래그를 읽어 짧은 웰컴 피드백 카드 노출(약 2.4초 후 자동 종료)
- `AuthStatusScreen` 공통 컴포넌트 추가
- `AuthRequired` 로딩 화면을 공통 카드 UI로 교체하여 인증 대기 화면 톤 통일
