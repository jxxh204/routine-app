## 📌 PR 요약
- 로그인 전에는 `/today`, `/calendar`, `/settings`, `/friends` 등 앱 주요 경로에 접근하지 못하도록 전역 가드를 추가했습니다.
- 변경 사항
  - `apps/web/src/middleware.ts` 추가
  - Supabase 세션 쿠키가 없으면 `/auth?next=...`로 즉시 리다이렉트
  - `/auth`, `/_next`, `/favicon.ico`는 공개 경로로 유지

## 🔎 문제 재현 (Before)
- [x] 재현 절차를 적었다
- [x] 기존 동작/에러를 캡처했다 (로그, 스크린샷, 영상 중 1개)

재현 절차:
1. 비로그인 상태에서 `/today` 접속
2. 클라이언트 렌더 이후에만 인증 체크가 걸려 초기 접근이 가능/깜빡임 발생 가능

기존 결과:
- 로그인 필요 정책이 클라이언트 레벨(`AuthRequired`)에만 있어, 서버 레벨에서 사전 차단이 부족함

## 🧠 원인
- 인증 차단이 페이지 컴포넌트 렌더 후(`useEffect`)에 동작함.
- 요청 단계에서 보호되지 않아 비로그인 접근을 선차단하지 못함.

## ✅ 해결 (After)
- Next 미들웨어에서 요청 단위로 세션 쿠키 유무를 확인해 비로그인 접근을 즉시 `/auth`로 리다이렉트.
- `next` 쿼리에 기존 경로를 보존해 로그인 후 원래 페이지로 복귀 가능.

변경 사항:
- `apps/web/src/middleware.ts` 신설
  - 공개 경로 예외 처리
  - 세션 쿠키(`sb-*-auth-token`) 체크
  - 미인증 시 `/auth?next=...` 리다이렉트
