## 📌 PR 요약
- 최신 main 웹 배포가 auth 페이지 빌드 오류로 실패해서, 배포 차단 원인을 제거했습니다.

기존 결과:

- 후속 로컬 빌드에서 `/auth` prerender 시 `useSearchParams() should be wrapped in a suspense boundary`

## 🧠 원인
- `supabase`가 nullable로 선언되어 있는데 null-safe 가드 이후에도 직접 참조 사용.
- App Router에서 `useSearchParams` 사용 컴포넌트에 Suspense 경계가 없어 prerender 단계에서 빌드 에러 발생.

## ✅ 해결 (After)
변경 사항:
- `apps/web/src/app/auth/page.tsx`
  - `const client = supabase; if (!client) return;` 가드 후 `client.auth.getSession()` 사용
  - 기존 페이지 로직을 `AuthPageContent`로 분리하고, default export에서 `<Suspense fallback={null}>`로 감쌈
