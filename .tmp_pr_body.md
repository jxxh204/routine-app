기존 결과:

## 🧠 원인
- 사진 데이터(base64)를 일일 스냅샷과 함께 localStorage에 저장
- localStorage 용량 제한 초과 시 저장 실패/부분 누락 발생
- 관련 코드/모듈/데이터 흐름:
  - `apps/web/src/app/today/today-view.tsx`
  - `routine-challenge-v1:YYYY-MM-DD` localStorage row

## ✅ 해결 (After)
- 인증 메타(doneByMe/doneAt)는 localStorage 유지, 사진 본문은 IndexedDB로 분리 저장
- Today/Calendar에서 `dateKey + routineId`로 이미지 복원
- 선택 이유: 사진 데이터는 대용량/바이너리 성격이라 localStorage보다 IndexedDB가 안정적

변경 사항:
- `apps/web/src/lib/proof-image-store.ts` 추가
- `today-view.tsx`의 사진 저장/복원 로직 변경
- `calendar/page.tsx`의 모달 이미지 복원 로직 추가
