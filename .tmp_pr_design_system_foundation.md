## 재현
- `docs/reference-ui/*` 기준으로 현재 web UI를 확인하면 다크 베이스+오렌지 포인트, 카드/버튼 위계가 공통 토큰 없이 페이지별 인라인 스타일에 분산되어 있음.
- `/auth`는 레이아웃/컬러/상태 UI가 단일 페이지 내부에 묶여 있어 다른 화면으로 확장 시 재사용이 어려운 상태였음.

## 원인
- 전역 토큰과 공통 컴포넌트 레이어 부재.
- 화면 구현 단위가 페이지 단위 인라인 스타일 중심이라 재사용성과 일관성 확보가 어려움.

## 해결
### 1) 디자인 토큰(CSS vars) 정리
- `apps/web/src/components/ui/design-system.css` 신설
  - 색상: background/surface/border/text/accent 계열 토큰 정의
  - 라운드/쉐도우/간격 토큰 정의
- `apps/web/src/app/globals.css`에서 신규 토큰을 import/alias하여 기존 변수(`--background`, `--foreground` 등)와 연결

### 2) 공통 컴포넌트 최소 세트 생성
- `apps/web/src/components/ui/design-system.tsx`
  - `PageShell`
  - `SectionHeader`
  - `AppCard`
  - `PrimaryButton`
  - `GhostButton`
  - `StatCard`
- `apps/web/src/components/ui/index.ts`에서 barrel export 구성

### 3) 컨셉 검증 페이지 1곳 우선 적용 (/auth)
- `apps/web/src/app/auth/page.tsx`를 신규 공통 컴포넌트 기반으로 리팩터링
- 기능 로직(소셜 로그인, 세션 체크, 오류/리다이렉트 흐름)은 유지하고 UI 계층만 교체
- 레퍼런스 톤(다크 + 오렌지 포인트 + 대문자 라벨/강한 헤드라인/라운드 카드)을 반영
