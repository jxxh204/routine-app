## 🧠 원인
- `useEffect(() => { void refresh(); }, [])`에서 호출된 `refresh()` 내부가 `setRows`, `setMessage` 등 state 업데이트를 수행.
- 최신 react-hooks lint rule(`set-state-in-effect`)이 effect 본문에서의 동기적 state 업데이트 경로를 에러로 판단해 CI를 fail 처리.
- 관련 코드: `apps/web/src/app/friends/page.tsx`

## ✅ 해결 (After)
- mount 시점 로드는 유지하되, effect 본문에서 직접 state 업데이트 흐름이 발생하지 않도록 타이머 큐로 이관.
- cleanup에서 타이머를 clear 하여 unmount 안전성 확보.

변경 사항:
- `useEffect` 내부를 아래처럼 변경
  - before: `void refresh();`
  - after: `const timer = window.setTimeout(() => { void refresh(); }, 0); return () => window.clearTimeout(timer);`
- 기능 동작(초기 목록 로드)은 유지
