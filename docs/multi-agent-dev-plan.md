# routine-app 멀티 에이전트 개발 운영안 (즉시 적용)

## 1) 역할 정의 (5개)

### 1. UX 에이전트
- 사용자 시나리오/플로우 정의
- 화면별 상태(기본/로딩/에러/빈 상태) 명세
- 사용성 리스크 사전 점검

### 2. UI 에이전트
- 페이지/섹션 UI 구현
- 반응형/접근성(키보드, aria) 적용
- 스크린샷 기반 시각 QA

### 3. 디자인컴포넌트 에이전트
- 공통 컴포넌트(Button, Card, Dialog 등) 개발/정비
- 토큰(색상/간격/타이포) 일관성 유지
- 스토리/예시 코드 관리

### 4. 비즈니스로직 에이전트
- 도메인 로직(시간대 인증, 상태 전이, 검증) 구현
- API/Supabase 연동, 예외 처리
- 테스트(단위/통합) 책임

### 5. 기획 에이전트
- 요구사항 우선순위(P0/P1/P2) 관리
- 수용 기준(AC), 배포 범위 확정
- 일정/리스크/의사결정 기록

---

## 2) 책임 매트릭스 (RACI)

| 작업 | UX | UI | 디자인컴포넌트 | 비즈니스로직 | 기획 |
|---|---|---|---|---|---|
| 요구사항/AC 확정 | C | I | I | C | A/R |
| 사용자 플로우/와이어 | A/R | C | I | I | C |
| 화면 구현 | C | A/R | C | C | I |
| 공통 컴포넌트 | C | C | A/R | I | I |
| 도메인 로직/API | I | C | I | A/R | C |
| 테스트 전략 | C | C | C | A/R | C |
| 릴리즈 결정 | C | C | I | C | A/R |

- A: 최종 책임, R: 실행 책임, C: 협의, I: 공유

---

## 3) 작업 흐름 (1일 단위)

1. **기획**: 오늘 목표 1~2개 확정 + AC 작성
2. **UX**: 플로우/상태 정의(문서 15분)
3. **디자인컴포넌트**: 필요한 공통요소 선반영
4. **UI + 비즈니스로직 병렬 작업**
5. **통합 QA**: 시나리오/에러/모바일 확인
6. **PR 생성 → 리뷰 → 머지**

규칙:
- 한 PR은 **하나의 사용자 가치**만 포함
- 컴포넌트 변경과 기능 변경이 크면 PR 분리

---

## 4) PR 규칙

### 브랜치
- 기능: `feat/<scope>-<short>`
- 버그: `fix/<scope>-<short>`

### PR 크기
- 권장: **300줄 내외**, 최대 600줄
- 초과 시 분리 PR 필수

### PR 본문 템플릿
- 목적(왜)
- 변경사항(무엇)
- 스크린샷/영상
- 테스트 결과(명령어 + 결과)
- 리스크/롤백 방법

### 리뷰 룰
- 최소 1명 승인
- 코멘트 해결 후 재요청
- CI 실패 시 머지 금지
