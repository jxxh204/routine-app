# Routine App Policy Rulebook

Last updated: 2026-04-15
Owner: @kimjaehwan5972

## 0) 목적
이 문서는 루틴앱의 제품/개발 운영 정책을 한곳에 고정 저장한다.
새 기능/수정/PR 작성 시 이 문서를 우선 기준으로 사용한다.

## 1) 인증/계정 정책
- 소셜 로그인 제공자: **Kakao + Apple만 지원**
- Google 로그인은 제공하지 않음
- 카카오 스코프는 실제 권한 보유 범위와 반드시 일치해야 함
  - 이메일 권한 없으면 `account_email` 요청 금지

## 2) 배포/브랜치 정책
- 자동 배포 임의 실행 금지 (요청 기반)
- 코드 변경 시 PR 필수
- 웹/앱 배포는 항상 **main 브랜치 기준**
- 사용자 리뷰 전까지 대기

## 3) PR 정책
- PR 템플릿 준수 필수
- 변경사항 스크린샷 첨부 필수
- 스크린샷은 **사용자에게 실제 표시되는 방식**으로 첨부
- 깨지는 링크(raw/blob 상대경로) 금지

## 4) 스크린샷 정책 (루틴앱 개발 채널)
- PR에만 두지 말고 채널에도 직접 업로드
- 이후 관련 작업마다 계속 적용
- **항상 모바일 화면 기준** 업로드

## 5) 개발 방식 정책
- 앱/웹/백엔드 모두 트랙별 TDD 적용
- 배포 관련 기술 선택/의사결정은 개발 채널 답변에서 제외
- 모바일은 현재 Expo 미사용 전제

## 6) 설정/운영 정책
- Supabase/Kakao/Apple OAuth 설정은 콘솔 설정값과 요청 스코프를 일치시킬 것
- 로그인 실패 시 사용자에게 원인(권한/스코프/설정)과 조치 가이드를 명시
- 인증 실패 후 UI가 무한 pending 상태에 빠지지 않도록 타임아웃/복구 처리 유지

## 7) 정책 변경 규칙
정책 변경 요청이 나오면 아래를 함께 남긴다.
- 변경일시
- 변경요청자
- 변경 전/후
- 적용 범위(웹/모바일/백엔드/운영)
- 관련 PR/커밋/이슈 링크
